# Codex Instructions

このリポジトリで数学教科書・問題集画像のJSON化に関わる作業を行う場合、常に `skills/textbook-math-extract/SKILL.md` の方針に従う。

必須方針:

- 出力JSONは `skills/textbook-math-extract/schema.json` で検証できる形にする。
- 数式はLaTeXで保持する。
- `original_text` は著作権保護のため必要最小限にする。
- 画像内に解答がない場合、推論した解答は許可するが `confidence` を下げる。
- スマホ学習アプリで使う前提で、`steps`、`hints`、`ui_components` を実用的に分解する。
- スクリプトは `skills/textbook-math-extract/scripts/` 配下を正とする。

## 公式・練習問題CSVのレビュー方針

`source/fomula.csv` と `source/problem.csv` を編集・レビューする場合は、次を確認する。

- `source/problem.csv` の `対象ID` は `source/fomula.csv` の `ID` に存在すること。
- `公式名` は対象IDの公式名と一致させる。表示用に別名が必要な場合でも、同期スクリプト側で公式マスタから引く設計を優先する。
- 問題は単独表示されるため、「同上」「上と同じ」など前後文脈に依存する表現を使わない。
- `数式（LaTeX）` は練習問題画面のヒントとして表示されるため、`E[X]`、`Var(X)`、`\sigma` だけのような抽象的な式にせず、問題条件を代入した途中式にする。
- 解答はアプリ上で選択トークンとして分解されるため、可能な限り簡潔な最終形にする。途中式や変数付き等式は `数式（LaTeX）` 側へ置く。
- 確率密度関数など同じ答えが続きやすい単元では、総確率の確認だけに偏らせず、区間確率、定数決定、期待値計算などに分散させる。
- 同一公式に対する問題群は、同じ数値入れ替えだけでなく、公式の使い方が分かれるようにする。
- 重要度の高い公式から優先して、公式ごとの問題数の偏りを減らす。
